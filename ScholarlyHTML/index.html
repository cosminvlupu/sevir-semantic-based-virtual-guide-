Abstract

Sevir or the semantic-based virtual guide is a travel-based web application which uses the user’s location and preferences to tailor suggested locales and tourist attractions to their liking and guide them through a tour of said locations.

Introduction

Sevir uses cartographic Web services, external travel-based APIs and social media integration alongside the user’s own stated preferences and needs to guide them through a tour of suggested locations appropriate to them

All data managed by the developed application is accessed via a SPARQL endpoint.

The app supports localization and internationalization in order to be usable by a wide variety of users.

Structure

The UI

The part of the application the user interacts with. Implemented as an Angular app which encapsulates all the frontend modules.

Register Module

Allows the user to create an account, either through social media integration or through the registration form. The data will be sent to the backend through an endpoint for the creation of the user’s account.

Login Module

Allows the user to log in through social media integration or through the login form. Using the appropriate endpoints the data will be compared to the user data and if correct will login the user in the application.

Social Media Integration

Using the public APIs of popular social Web applications such as Google, Facebook or Twitter allows the user to login through their already existing account or link an account with the application and allows them to post to said social Web application.

Backend

A Spring app that encapsulates and handles all the backend modules.

REST Controller

The controller part of the REST API which manages the user data. Using the /register, /login and /logout endpoints the REST controller communicates with the frontend.

User Service

A backend service that handles the user data and the logic associated with it. Assures the communication between the REST API and the database.

Database

AStores the user’s personal data, preferences and geographical data in a way that facilitates easy retrieval and storing of new data.

Browse Location

Using the user’s location, preferences and the geographical data suggests to the user a list of relevant, close locales and other tourist attractions and allows them to choose one or multiple of them and, once they do, guides them to said location or locations as a tour.

Location Service

A backend service that handles the location data and the logic associated with it. Manages the external travel-related APIs.

External APIs

The location and geographical data used by the application is collected through external travel-related APIs such the OpenStreetMap API, the LinkedGeoData API and other external service APIs.

Conlusions

Sevir’s documentation and implementation are conceptualized in a way which has the purpose to allow the application to be easy to use and understand while making sure it accomplishes all of its stated goals.
